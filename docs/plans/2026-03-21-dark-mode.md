# Dark Mode

**Goal:** Add Flexoki dark mode (same palette as stephango.com) with a toggle button and system-preference fallback.

**Approach:** Define semantic CSS variables that flip between light and dark Flexoki values. Use Tailwind v4's `dark:` variant (class strategy) so a `.dark` class on `<html>` controls theming, with `prefers-color-scheme` as the default. Existing `rurikon-*` tokens already contain all the dark palette values — no new colors needed, just semantic mappings. A small client component handles the toggle and persists preference to localStorage.

**Files touched:**
- `app/globals.css` — semantic CSS variable declarations for light/dark; dark Shiki token overrides
- `app/layout.tsx` — update `colorScheme` viewport, add `dark:` class to html/body where needed
- `components/navbar.tsx` — replace hardcoded `rurikon-*` classes with `dark:` variants; add theme toggle button
- `components/theme-toggle.tsx` — new client component: reads/writes `localStorage`, applies `.dark` class to `<html>`
